# এডমিন প্যানেল সেটআপ গাইড

এই প্রজেক্টে এখন একটা প্রোটেক্টেড এডমিন প্যানেল যুক্ত করা হয়েছে (`/admin` রুটে), যেখান থেকে
পোর্টফোলিওর সব ডেটা (নাম, বায়ো, শিক্ষাজীবন, লেখালেখি, যোগাযোগ তথ্য ইত্যাদি) ব্রাউজার থেকেই
এডিট করা যাবে — কোনো কোড টাচ না করে।

## ১. Supabase প্রজেক্ট বানান
1. https://supabase.com -এ লগইন করুন, নতুন প্রজেক্ট তৈরি করুন।
2. প্রজেক্ট ড্যাশবোর্ডে **Settings → API** থেকে `Project URL` এবং `anon public key` কপি করুন।

## ২. ডাটাবেস টেবিল বানান
1. Supabase ড্যাশবোর্ডে **SQL Editor** খুলুন।
2. এই প্রজেক্টের `supabase_setup.sql` ফাইলের পুরো কন্টেন্ট কপি-পেস্ট করে **Run** করুন।
   এটা `portfolio_content` টেবিল ও প্রয়োজনীয় সিকিউরিটি (RLS) পলিসি বানাবে।

## ৩. এডমিন ইউজার বানান
1. Supabase ড্যাশবোর্ডে **Authentication → Users → Add user** ক্লিক করুন।
2. আপনার ইমেইল ও একটা পাসওয়ার্ড দিয়ে ইউজার তৈরি করুন।
3. এই ইমেইল/পাসওয়ার্ড দিয়েই `/admin` পেজে লগইন করবেন।

## ৪. .env ফাইল বানান
প্রজেক্টের রুটে `.env` নামে একটা ফাইল বানান (`.env.example` দেখে কপি করতে পারেন):

```
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-public-key
```

## ৫. আপনার আসল ডেটা Supabase-এ পাঠান (একবার)
`src/data/portfolioData.js` ফাইলে আপনার আসল তথ্য (নাম, ফোন, ইমেইল, শিক্ষাজীবন ইত্যাদি)
আগে থেকেই লেখা আছে। এটা একবারে Supabase-এ পাঠাতে:

1. `.env` ফাইলে আরেকটা লাইন যুক্ত করুন (Supabase Dashboard → Settings → API → `service_role` key):
   ```
   SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
   ```
2. রান করুন:
   ```
   node scripts/seed.js
   ```
   এটা `portfolioData.js`-এর সব তথ্য একবারে Supabase টেবিলে বসিয়ে দেবে।
3. **এই কাজ শেষে `.env` থেকে `SUPABASE_SERVICE_ROLE_KEY` লাইনটা মুছে ফেলুন** —
   এই key কখনো লাইভ সার্ভারে/গিটে রাখা ঠিক না, এটা শুধু একবার সিড করার জন্য।

এর পর থেকে যা এডিট করতে চান, `/admin` প্যানেল থেকেই করবেন।

## ৬. রান ও বিল্ড
```
npm install
npm run dev      # লোকালি টেস্ট করার জন্য
npm run build    # প্রোডাকশন বিল্ড
```

ওয়েবসাইট দেখতে: `http://localhost:5173/`
এডমিন প্যানেল: `http://localhost:5173/admin`

## ৭. হোস্টিং
Netlify বা Vercel-এ ডিপ্লয় করলে `/admin` রুট কাজ করার জন্য `_redirects` (Netlify) এবং
`vercel.json` (Vercel) ফাইল ইতিমধ্যে যুক্ত করা আছে। ডিপ্লয় করার সময় Environment Variables-এ
`VITE_SUPABASE_URL` ও `VITE_SUPABASE_ANON_KEY` যুক্ত করতে ভুলবেন না।

## কীভাবে কাজ করে
- ওয়েবসাইট প্রথমে Supabase থেকে ডেটা লোড করার চেষ্টা করে।
- যদি `.env` সেট না থাকে, Supabase কানেকশন ব্যর্থ হয়, বা কোনো সেকশনের ডেটা Supabase-এ
  না থাকে — তাহলে সাইট ভেঙে না গিয়ে **ডেমো/প্লেসহোল্ডার ডেটা** দেখাবে
  (যেমন "আপনার নাম", "01XXX-XXXXXX", "youremail@example.com") — `src/data/demoData.js` ফাইলে।
  এতে ভুলবশত আসল তথ্য পাবলিকলি দেখা যাওয়ার সুযোগ থাকে না, যতক্ষণ না Supabase ঠিকভাবে
  কনফিগার করা হয়।
- `/admin` পেজে লগইন করার পর প্রতিটা সেকশন (ব্যক্তিগত তথ্য, শিক্ষাজীবন, লেখালেখি ইত্যাদি)
  আলাদা ট্যাবে এডিট করা যায়। "সেভ করুন" চাপলে সরাসরি Supabase-এ আপডেট হয় এবং সাইট
  রিফ্রেশ দিলেই নতুন ডেটা দেখা যাবে — রিডিপ্লয় করার প্রয়োজন নেই।
